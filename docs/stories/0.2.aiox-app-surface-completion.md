## Story
**As a** maintainer da OpenF1 Telemetry Platform,
**I want** completar as páginas prometidas pela navegação, endurecer o fluxo de predictions e elevar a UX/UI do app,
**so that** a superfície do produto fique coerente com as integrações já existentes no backend e pronta para staging avançado.

## Acceptance Criteria
1. A navegação do app aponta apenas para páginas realmente implementadas.
2. As rotas `Telemetry`, `Drivers`, `Circuits`, `Settings` e `Help` passam a existir e usam APIs já integradas no projeto.
3. A experiência de predictions usa dados reais e expõe claramente o estado do modelo/pipeline quando aplicável.
4. O app reduz estados quebrados de UX visíveis hoje, incluindo links mortos, placeholders técnicos e feedback insuficiente.
5. `npm run lint`, `npm run typecheck`, `npm test` e `npm run build` passam ao final.
